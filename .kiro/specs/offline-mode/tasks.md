## 1. オフライン機能の基礎実装 (1.2, 2.1)
- [x] 1.1 (P) オフライン設定値の永続化と取得ロジック
  - アプリ起動および設定変更の際に、オフライン状態のフラグがローカルストレージ（UserDefaults）へ即座に読み書きされること。
  - アプリを再起動した際に、前回適用されていたオフラインモードの設定が正常に復元・適用されること。
  - _Requirements: 1.2_
  - _Boundary: AuthViewModel_
- [x] 1.2 (P) 保存済みコミックデータのUI表示用モデルへの変換機能
  - ローカルのメタデータから取得したダウンロード済みのコミック（LocalComic）を、既存の一覧表示要素（DriveItem）へ正常に変換できること。
  - 表紙画像のローカル絶対パスがサムネイルURLとして正しくマッピングされること。
  - _Requirements: 2.1_
  - _Boundary: DriveItem_

## 2. ライブラリと表示制御の実装 (2.1, 2.2)
- [x] 2.1 オフラインモード時のライブラリデータ取得処理の制御
  - オフラインフラグが有効な場合、Google Drive API通信を完全にバイパスし、ローカルに保存されている漫画リストのみを表示データとして読み出すこと。
  - オフライン表示時は、次ページトークンを無効化して追加読み込み処理が発生しないようにすること。
  - _Requirements: 2.1, 2.2_
  - _Boundary: LibraryViewModel_
  - _Depends: 1.1, 1.2_

## 3. UIの統合と機能制限 (1.1, 1.2, 1.3, 3.1, 3.2, 3.3)
- [ ] 3.1 アプリ起動時の認証バイパス制御
  - オフラインモードが有効な状態でアプリを起動した場合に、Googleログインの成功有無にかかわらず、ログイン画面をスキップして直接ライブラリ画面に遷移すること。
  - _Requirements: 1.2_
  - _Boundary: App_
  - _Depends: 1.1_
- [ ] 3.2 (P) ログイン画面におけるオフライン導線の追加
  - Googleログイン画面に「オフラインで利用」ボタンを配置し、タップすることで認証処理を行わずに直接ライブラリ画面に進めること。
  - _Requirements: 1.1_
  - _Boundary: LoginView_
  - _Depends: 1.1_
- [ ] 3.3 ライブラリ画面におけるオフライン制御UIの追加
  - ライブラリ画面にオフライン切り替えスイッチ（トグル）を配置し、オン/オフ操作に応じてリスト表示が即座に切り替わること。
  - オフライントグルがONの際、画面上部に「オフラインモード」を示す明確なインジケータが表示され、新規ダウンロード操作UI（ボタンや一括ダウンロードシート遷移）が非活性または非表示になること。
  - _Requirements: 1.1, 1.3, 3.3_
  - _Boundary: LibraryView_
  - _Depends: 2.1_
- [ ] 3.4 オフライン時の閲覧と閲覧制限の制御
  - ダウンロード済みの漫画をタップした際、ローカルのデータソース（LocalComicSource）を通じてビューアーが正常に起動し読書できること。
  - 未ダウンロードのアイテムをタップした際、またはオフライン時に開けない動作を試みた際に、エラーメッセージが表示されて起動がブロックされること。
  - _Requirements: 3.1, 3.2_
  - _Boundary: LibraryView_
  - _Depends: 3.3_

  - トグルのON/OFFにより、未ダウンロードアイテムが瞬時に非表示化および表示化されることを実証すること。
